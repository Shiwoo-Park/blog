# 이미지 가이드

블로그 포스트에 사용할 이미지에 대한 가이드입니다.

## Open Graph 이미지 (og:image)

### 권장 사이즈

**표준 사이즈**: 1200 x 630px (1.91:1 비율)

이 사이즈는 다음 플랫폼에서 최적화되어 있습니다:
- Facebook: 1200 x 630px
- Twitter: 1200 x 630px (Card)
- LinkedIn: 1200 x 627px
- 카카오톡: 1200 x 630px (한국 특화)

### 최소/최대 사이즈

- **최소**: 600 x 315px
- **최대**: 1200 x 630px (권장)
- **파일 크기**: 1MB 이하 권장 (로딩 속도 고려)

### 파일 형식

- **PNG**: 투명 배경이 필요한 경우
- **JPG**: 일반적인 경우 (파일 크기가 작음)
- **WebP**: 최신 브라우저 지원 시 (가장 작은 파일 크기)

## 카테고리별 타이틀 배경 이미지

각 카테고리(기본, 개발, 투자, 여행)별로 대표 이미지를 만드는 것을 권장합니다.

### 이미지 구성 요소

1. **배경**: 카테고리 컨셉에 맞는 배경
2. **제목 영역**: 포스트 제목이 들어갈 영역 (중앙 또는 상단)
3. **카테고리 표시**: 작은 배지나 텍스트로 카테고리 표시
4. **브랜드 컬러**: 블로그의 일관된 색상 사용

### 카테고리별 추천 컨셉

#### 💻 개발 (dev)
- **컬러**: 파란색 계열 (#667eea, #764ba2)
- **이미지 요소**: 코드, 터미널, 개발 도구 아이콘
- **분위기**: 기술적, 전문적
- **예시**: 어두운 배경에 네온 코드 라인, 터미널 창

#### 📈 투자 (invest)
- **컬러**: 초록/금색 계열 (#10b981, #f59e0b)
- **이미지 요소**: 차트, 그래프, 주식 시장 관련 아이콘
- **분위기**: 성장, 안정
- **예시**: 상승하는 차트, 금융 그래프

#### ✈️ 여행 (travel)
- **컬러**: 따뜻한 톤 (#f97316, #ec4899)
- **이미지 요소**: 비행기, 지도, 여행지 풍경
- **분위기**: 모험, 자유
- **예시**: 여행지 풍경, 지도, 비행기 실루엣

#### 🏠 기본 (home)
- **컬러**: 중립적 톤 (#6b7280, #9ca3af)
- **이미지 요소**: 간단한 기하학적 패턴
- **분위기**: 깔끔, 미니멀
- **예시**: 그라데이션, 기하학적 패턴

### 이미지 파일 구조 제안

```
resources/
├── og-default.png      # 기본/홈 이미지
├── og-dev.png          # 개발 카테고리 이미지
├── og-invest.png       # 투자 카테고리 이미지
└── og-travel.png       # 여행 카테고리 이미지
```

### 이미지 생성 도구 추천

1. **Canva**: 템플릿 기반, 쉬운 사용
   - 사이즈: 1200 x 630px 템플릿 사용
   - 무료 플랜으로도 충분

2. **Figma**: 디자인 툴, 더 정교한 제작 가능
   - 무료 플랜 사용 가능

3. **Photoshop/GIMP**: 전문 디자인 도구

4. **온라인 OG 이미지 생성기**:
   - [og-image.vercel.app](https://og-image.vercel.app/)
   - [Bannerbear](https://www.bannerbear.com/)

### 이미지 사용 방법

#### 방법 1: 카테고리별 기본 이미지 사용

`_config.yml`에 카테고리별 기본 이미지 설정:

```yaml
# 카테고리별 기본 이미지
category_images:
  dev: "/resources/og-dev.png"
  invest: "/resources/og-invest.png"
  travel: "/resources/og-travel.png"
  default: "/resources/og-default.png"
```

레이아웃에서 자동으로 적용하도록 수정 가능

#### 방법 2: 포스트별 개별 이미지

각 포스트의 frontmatter에 직접 지정:

```yaml
---
layout: post
title: "제목"
image: "/resources/og-invest.png"  # 또는 포스트별 고유 이미지
---
```

#### 방법 3: 동적 이미지 생성 (고급)

제목을 포함한 동적 이미지를 생성하는 방법 (서버 사이드 필요)

### 템플릿에서 이미지 필드 사용 가이드

현재 템플릿의 `image` 필드는 다음과 같이 사용합니다:

```yaml
---
layout: post
title: "제목"
image: "/resources/og-invest.png"  # 카테고리별 이미지 경로
---
```

**권장 사용법**:
- 카테고리별 기본 이미지 사용: `/resources/og-{category}.png`
- 특별한 포스트는 개별 이미지 생성 가능
- 이미지가 없으면 jekyll-seo-tag가 기본값 사용

## 이미지 최적화 팁

1. **압축**: TinyPNG, ImageOptim 등으로 압축
2. **WebP 변환**: 최신 브라우저 지원 시 WebP 사용
3. **Lazy Loading**: 이미지 로딩 최적화
4. **CDN 사용**: 이미지 전송 속도 개선
