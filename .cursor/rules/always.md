# Cursor Always Rules

이 파일의 규칙은 모든 프롬프트와 작업에 항상 적용됩니다.

## 📝 문서 작성 규칙

### 1. 문서 구성 및 어조

- **요약형 구성**: 핵심 내용을 간결하고 명확하게 전달
- **어조**: 전문적이면서도 이해하기 쉬운 설명체 사용
- **구조**: 개념 → 사용법 → 실무 예시 → 정리 순서로 구성 권장
- **불필요한 장식어나 과도한 설명 지양**

### 2. 블로그 포스팅 템플릿 준수

신규 블로그 포스팅 작성 시 반드시 `_sample.md` 파일의 템플릿 양식을 따라야 합니다:

```markdown
# 제목

> 날짜: YYYY-MM-DD

[목록으로](https://shiwoo-park.github.io/blog)

---

내용이 여기에 옵니다

---

[목록으로](https://shiwoo-park.github.io/blog)
```

**템플릿 준수 사항:**

- 제목은 `#` 하나만 사용 (H1)
- 날짜는 `> 날짜: YYYY-MM-DD` 형식으로 작성
- 상단과 하단에 목록 링크 포함
- 섹션 구분은 `---` 사용

### 3. 신규 포스팅 작성 시 자동 처리

신규 블로그 포스팅 파일을 생성할 때 다음을 자동으로 수행:

1. **날짜 정보 자동 입력**

   - 현재 날짜를 `YYYY-MM-DD` 형식으로 자동 입력
   - 예: `> 날짜: 2025-11-11`

2. **README.md 자동 업데이트**
   - 프로젝트 루트의 `README.md` 파일에 새 포스팅 링크 추가
   - 해당 연도/월 섹션(예: `### 2025.11`)이 있으면 그 섹션의 **최상단**에 추가
   - 해당 연도/월 섹션이 없으면 새로 생성하여 추가
   - 링크 형식: `- [제목](상대경로)`
   - 날짜 섹션 형식: `### YYYY.MM` (점으로 구분)

**README.md 업데이트 예시:**

```markdown
### 2025.11

- [새로운 포스팅 제목](dev/category/new_post.md) ← 최상단에 추가
- [Git Worktree: 여러 브랜치를 동시에 작업하는 방법](dev/ai/git_worktree.md)
- [개발자에게 유용한 AI 프롬프트](dev/ai/useful_prompts.md)
```

### 4. 파일 경로 규칙

- 포스팅 파일은 적절한 카테고리 디렉터리 하위에 생성
- 예: `dev/ai/`, `dev/python/`, `dev/aws/` 등
- 파일명은 소문자와 언더스코어 사용 권장 (예: `git_worktree.md`)

### 5. 기존 포스팅 수정 시

- 기존 포스팅 파일을 수정할 때는 날짜를 변경하지 않음
- README.md는 수정하지 않음 (링크는 이미 존재하므로)

## 🔍 작업 시 확인 사항

문서 작업 시 다음을 확인:

1. ✅ 템플릿 양식 준수 여부
2. ✅ 날짜 형식 정확성 (YYYY-MM-DD)
3. ✅ README.md 링크 추가 여부 (신규 포스팅만)
4. ✅ 링크 경로 정확성
5. ✅ 문서 어조 및 구성 일관성
