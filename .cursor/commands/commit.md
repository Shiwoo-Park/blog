# 코드 커밋 & 푸시

다음 순서로 작업을 수행해줘:

1. 현재 상태에서 git 수정사항 확인
2. 변경사항 기반으로 간결하게 커밋메시지 작성
3. 커밋메시지에 종류별 prefix 생성
  - `dev`: 사이트 개발코드 관련 변경사항 발생 시
  - `post`: 블로그 포스팅 관련 변경사항 발생 시
  - `etc`: 위 둘중 어느하나에도 속하지 않는 변경사항 발생 시
4. 커밋메시지 작성된것을 요청자에게 확인받기
5. 보완이 필요한경우 내용을 보완하여 다시 확인받기 (반복)
6. "커밋", "진행", "OK", "확인" 등 요청자에게 메시지를 확정받으면 그대로 커밋

## 1줄 커밋메시지 예시 

- `post: 개발 블로그 - Cursor IDE 토큰 사용량 절약 가이드 포스팅 추가`
- `post: 여행 블로그 - 토스 체크카드로 태국바트 인출 방법 포스팅 수정`
- `dev: 메인페이지 블로그 소개문구 변경 및 메타 태그 추가`
- `dev: Google Analytics 트래킹 추가`
- `etc: Cursor 용 AI 프롬프트 수정`

## 여러줄 커밋메시지 예시

```
post: 블로그 포스팅 추가/수정

- 개발 블로그 - Cursor IDE 토큰 사용량 절약 가이드 포스팅 추가
- 여행 블로그 - 토스 체크카드로 태국바트 인출 방법 포스팅 수정
```
