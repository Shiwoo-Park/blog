# 포스팅 작성 요청

언급한 주제 또는 요청 내용에 대하여 블로그 포스팅을 작성해줘

포스팅 작성 시, 아래 사항들을 준수하도록 해줘

## 포스팅 작성 단계

1. 주제 명확화: 제시한 주제에 대해 구체화가 필요하거나 질문이 있으면 질문하기
2. 글의 단락별 주제를 요약하여 표시: 어떤식으로 글의 내용이 구성될 것인지 한눈에 확인하고 검토할 수 있도록 표시
3. 글의 전체 구성에 대한 피드백 -> 반영 반복: 2번에 대한 피드백을 받아서 수정된 사항으로 다시 2번으로 돌아가고 "글 작성" 해달라는 말이 나올때까지 반복
4. 글 작성: 요청자가 "글 작성" 해달라고 했을때만 이 단계에 들어갈 수 있음. 계획했던 글의 구성에 따라 내용을 생성

## 글 작성 시, 원칙

- `posts\_post_template.md` 양식을 따라 글을 작성하도록 할것.
- 포스팅 작성 시, 날짜는 반드시 오늘 날짜를 기입할것.
- 포스팅 작성 후, 해당 포스팅의 소속 카테고리에 따라서 아래 3개 파일중 하나에 작성날짜에에 따른 포스팅 링크를 해당 년월의 최상단에 입력할것
  - 개발 블로그: `posts/dev/home.md`
  - 투자 블로그: `posts/invest/home.md`
  - 여행 블로그: `posts/travel/home.md`
  - (ex) `posts/dev/*/my_post.md` 일 경우 `posts/dev/home.md` 에 추가
  - 코드 스니펫의 경우: `snippets/home.md`

## 포스팅 작성 방식

1. 되도록 너무 장황하지 않게 핵심만 간결하게 작성
2. 특정 지식에 대한 설명을 할 경우, 해당 주제를 잘 모르는 사람도 쉽게 알아들을 수 있도록 작성
3. 특별히 내용이 복잡하거나 이해하기 어려운 경우, 간단한 예시를 곁들여 이해를 돕도록 할 것
4. 깊이있는 개념을 설명해야하는 경우, 가장 기본이 되는 지식부터 점진적으로 핵심만 간단하게라도 추가하여 이해를 돕기.
5. 맨 마지막에는 전체 핵심내용만을 아주 간단하게 요약한 **핵심요약** 섹션을 별도로 구성
